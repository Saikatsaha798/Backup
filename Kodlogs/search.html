Searching (Explanation) #

Searching i.e., checking if value is present in list is solved by following ways :

1. Linear Search: #
This is the bruteforce and easiest way for searching but for the simplicity we get in return of more time complexity. This involves traversing the whole list and checking if any element matches to our searching term.

Complexity: O(n)

Algorithm:
Step 1: Set found = 0 and repeat step 2, i for all elements in list.
Step 2: Check if i == item then item is found and set found = 1 and print Item was found in list and break.
[End of Step 1 Loop]
Step 3: Check if found == 0 print Item was not found in list.

2. Binary Search: #
This is a bit tricky but in return we get fast and less time complexity. This basically works in SORTED ARRAY and this basically is one of the disadvantages. This takes the mid term and checks with the searching term and figures out whether it matches if yes, then Voila you got the number otherwise just takes left of the list, if the searched term is smaller and right if larger. And carries on until finds the number or no part of list is left.

Complexity: O(n)

Algorithm:
Step 1: Check if list is not empty then,
[Inside of if]
Step 2: Check if mid term of the list is equal to the item then print Item was found in list.
Step 3: Else if item is bigger than mid term then perform step 1,2,3,4 with right part of the list.
Step 4: Else if item is smaller then mid term then perform step 1,2,3,4 with left part of the list.
[End of Step 1 if]
Step 5: Else print Item was not found in list.

Applying logic in Code #

Lots of explanation and theory, but let's see those in code. Basically, we will use python.

Linear Search #

def linear_search(l, a):
    found = 0
    for i in l:
        if a == i:
            found = 1
            return True
    if found == 0:
        return False

It is a function, which takes list as l and item as a in parameters and performs Linear Search over the list and returns True if found and False else. So, it basically runs variable i for each elements in list l, then checks if i and item a is equal and if those are equal found becomes and 1 and True is returned. After, traversing through the whole list and if item is not found, then found remains at its initial value that is 0 and then False is returned. Hence, we successfully executed the logic of Linear Search in a piece of Python function.

Binary Search #

def binary_search(l, a):
    res = False
    if len(l) != 0:
        mid = l[(len(l)//2)]
        if mid == a:
            return True
        elif mid > a:
            res = binary_search(l[:(len(l)//2)], a)
        else:
            res = binary_search(l[(len(l)//2)+1:], a)
        return res
    else:
        return res

It is a function, which takes sorted list as l and item as a in parameters and performs Binary Search over the list and returns True if found and False else. So, it basically starts with checking if list is empty then it starts with finding the middle most element as storing it in variable mid and checks if that is the element we are searching for, if so then it returns True else, continues to check if the item a is bigger or smaller than variable mid. If mid is smaller than a, then right part of the list is taken into consideration and the same process is repeated and if bigger, then done the same with left part of the list.
